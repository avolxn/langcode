# LangCode: Python-клон OpenCode

## Миссия
Создать Python-клон OpenCode CLI. Каждая функция, промпт, инструмент и поведение должны точно соответствовать оригиналу. Никаких упрощений, никаких интерпретаций.

## Источник
- **Путь:** `./opencode/` (TypeScript/JavaScript)
- **Цель:** Python 3.11+ с async/await

## Технологический стек (обязательно)
- **Agent Framework:** LangChain
- **CLI:** Typer + Rich
- **API:** FastAPI + Uvicorn
- **Storage:** SQLAlchemy (async) + SQLite
- **Package Manager:** uv

## Непреложные правила
1. **Точность прежде всего:** Сначала точно воспроизвести поведение OpenCode, потом оптимизировать
2. **Промпты:** Копировать слово в слово, сохранять всё форматирование
3. **Инструменты:** Идентичные сигнатуры, выводы и обработка ошибок
4. **Конфигурация:** Совместимый формат или предоставить скрипт миграции
5. **Модульность:** Отделить основную логику от слоёв CLI/UI

## Принципы работы

**Использование инструментов:**
- Использовать параллельные вызовы инструментов для независимых операций
- Делегировать большие исследования Explore agent
- Использовать Plan agent для сложного дизайна модулей
- Держать основной контекст чистым, использовать subagents для глубоких погружений

**Качество:**
- Запускать `/simplify` после каждого модуля
- Писать тесты параллельно с реализацией
- Постоянно валидировать относительно OpenCode

**Коммуникация:**
- Краткие обновления после каждого модуля
- Немедленная эскалация блокеров или неопределённостей
- Документировать архитектурные решения в `docs/DECISIONS.md`

**Критичные правила:**
- Не переходить к следующему этапу без одобрения пользователя
- Тестировать каждый модуль относительно OpenCode перед переходом дальше
- Спрашивать при неуверенности, никогда не предполагать
- Вести заметки о миграции для всех отклонений
